# Index Study Lab v1.0.0

Release date: 2026-05-17

## Why This Is V1

V1 is the first justified baseline for future development. The app is not
"finished"; it is now mature enough that future work can build on explicit
contracts instead of chat memory, screen scraping, or informal assumptions.

The important V1 achievement is trust infrastructure:

- studies run through a shared route and settings model
- source policy and return-basis metadata are explicit
- unsafe TRI proxy behavior is blocked instead of quietly tolerated
- metric presentation is governed by a cross-study registry
- completed runs have a durable backend ledger
- options evidence has a stored screener archive and tracked straddle ledger
- the assistant boundary is versioned, deterministic, and non-executing
- live AI drafting is visible only as an experimental StudyPlan draft flow

## Product Surface Included

- Index studies: risk-adjusted return, relative comparison, seasonality, rolling
  returns, SIP simulator, lumpsum versus SIP, drawdown, and sector snapshot.
- Options workflow: monthly straddle, options screener, options validation, and
  trade-validation backend path for tracked straddle outcomes.
- App-level settings: automations/runtime health, run history, and Study Builder.
- Symbol discovery: local catalog, remembered/manual symbols, Finnhub-backed
  discovery helpers, and yfinance fallback paths.
- Assistant-safe foundations: StudyPlan validation, intent planning, recipe
  storage, assistant readiness, run-brief handoff, explanation brief, and
  experimental live AI StudyPlan drafting.

## Evidence And Guardrails

- `POST /api/assistant/study-plan-live-draft` lets a model draft only
  `study-plan-v1` JSON.
- `#settings/study-builder` exposes `Experimental: Live AI Draft`, but the app
  still validates the draft and never executes a study from that action.
- `POST /api/assistant/study-run-brief` is the backend-owned entrypoint for
  future run explanations.
- The metric registry controls headline, diagnostic, suppressed, and export
  behavior so future assistant work does not invent metric authority.
- Options validation and trade validation are separated: underlying validation
  remains a signal-hypothesis check, while tracked straddles form the
  trade-outcome evidence path.

## Known Boundaries

- No approved true total-return source is configured. TRI-labeled runs are
  correctly blocked until that sourcing decision is solved.
- Live AI is limited to StudyPlan drafting. It does not execute studies and does
  not generate result prose.
- Result explanation is still contract-first: the durable run brief and
  explanation permission envelope exist, but free-form prose should remain a
  later controlled slice.
- Options calendar trades are still screening hypotheses. V1 trade-aware
  evidence intentionally tracks only front-month long and short straddles.
- This is a local-first research tool, not a production trading system and not
  investment advice.
